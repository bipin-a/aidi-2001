# Assignment: MCPs and Skills Across the Software Lifecycle

## Goal

In this assignment, you will:

* set up **MCP servers** across multiple AI coding platforms
* install and use **skills** in a small project
* demonstrate your workflow in a **video**
* submit a **GitHub repository** that shows your process

The purpose is to help you learn how MCPs and skills can support real software work across planning, development, testing, and architecture.

# Part 2: Skills Setup

Set up skills in:

* **Claude Code**
* **Codex**

## Required skills

You must install and use these 5 skills from Matt Pocock’s collection:

[https://github.com/mattpocock/skills/tree/main](https://github.com/mattpocock/skills/tree/main)

* `grill-me`
* `write-a-prd`
* `prd-to-issues`
* `tdd`
* `improve-codebase-architecture`

## What these skills should help you do

These skills are intended to support a realistic software workflow:

* `grill-me` → challenge and strengthen your plan
* `write-a-prd` → define what you are building
* `prd-to-issues` → turn the PRD into GitHub Issues
* `tdd` → build using a test-driven workflow
* `improve-codebase-architecture` → review and improve the codebase structure

## What you need to show

In your video, you must show:

* that the skills are installed in Claude Code and Codex
* where and how you called each skill
* what output each skill produced

# Part 3: Project Demonstration

Create a **small, simple proof-of-concept project** in a **separate GitHub repository**.

## Project requirement

Your project should be simple enough to complete for this assignment. The goal is to demonstrate your **workflow**, not to build a large or complex product.

## Suggested project ideas

Choose something small, such as:

* a simple to-do app
* a habit tracker
* a note-taking app
* a bookmark manager
* a basic expense splitter
* a small API with 1–2 endpoints
* a weather or quote dashboard
* a timer or pomodoro app

## Required workflow

Use the skills in this order:

1. `grill-me`
2. `write-a-prd`
3. `prd-to-issues`
4. `tdd`
5. `improve-codebase-architecture`

## What your repo must show

Your GitHub repository should include:

* the project code
* issues created from your workflow
* a final working version of the project

## What you must show in the video

Show all of the following:

* the final working project
* the GitHub repository
* the GitHub Issues created through the skill workflow
* your chat history showing where you used each required skill
